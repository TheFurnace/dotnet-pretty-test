Simple developer workflow

1. Make changes
   - Edit code or docs in the repository.

2. Build
   - Ensure the project compiles before adding tests.
   - Example: dotnet build

3. Write tests (if needed)
   - Add or update unit/integration tests to cover changes.

4. Run tests
   - Run the test suite and fix failures.
   - Example: dotnet test

5. Commit changes
   - Stage and commit only the relevant files with a clear message.
   - Example: git add -A && git commit -m "Short, clear summary of changes"

Notes
- Keep commits small and focused.
- Run the build and tests locally before pushing.
